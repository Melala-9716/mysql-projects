## 🏫 Biruk Academy School Management Database

### Description
A multi-branch school management system designed to handle students, teachers, payments, grades, and schedules using a normalized relational database structure.

---

### Database Modules
- Branch Management
- Student Records
- Teacher Records
- Payments System
- Grade Reports
- Class Scheduling

---

### Features
- Multi-branch architecture
- Student–teacher relational mapping
- Payment tracking (Paid / Partial / Overdue)
- Academic performance tracking
- Timetable scheduling system
- Aggregated SQL reports using JOINs and GROUP BY

## 🛒 E-Commerce Management System

### Description
A relational database system designed to manage online shopping operations including customers, products, categories, orders, payments, and shipping information using MySQL.

---

### Database Modules
- Customer Management
- Product Management
- Category Management
- Order Processing System
- Order Item Management
- Payment Management
- Shipping Management

---

### Features
- Customer registration and profile management
- Product inventory and stock tracking
- Product categorization system
- Order creation and order status tracking
- Multiple products per order using OrderItems relationship
- Payment recording and transaction tracking
- Shipping and delivery status management
- Sales revenue calculation
- Relational integrity using primary keys and foreign keys

---

### ER Diagram

![E-Commerce Management ER Diagram](./diagrams/E-Commerce%20diagram.jpg)

---

### Sample Queries Included
- Display all customers and products
- Find available products
- Find products that are out of stock
- View customer order history using JOIN
- Display products included in each order
- Generate customer purchase reports
- Show payment details for orders
- Track shipping information
- Display products with their categories
- Calculate total sales revenue
- Count total orders per customer
- Find the highest-priced product
- Find low-stock products
- Show pending payments
- Generate complete customer invoices using multiple JOIN operations
